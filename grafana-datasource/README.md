# Tuist data source for Grafana

A Grafana data source plugin that turns Tuist build and test duration metrics
into Grafana time series. It is a thin client over the Tuist server's public
metrics API; all aggregation (p50/p90/p99, time bucketing) happens server-side.

## What you can query

- **Build durations** and **Test durations**, each as `average`, `p50`, `p90`,
  and `p99` series over the dashboard's time range.
- Filters: CI vs local, scheme, and (for builds) configuration.
- Projects, schemes, and configurations populate from the server, so they also
  work as dashboard template variables.

## Configuration

| Field | Stored as | Notes |
| --- | --- | --- |
| Server URL | `jsonData.url` | Defaults to `https://tuist.dev`. |
| Account token | `secureJsonData.apiToken` | Encrypted at rest; never sent to the browser. |

The account token needs `project:builds:read` and `project:tests:read` scopes
(both included in the `mcp` scope group) and access to the relevant projects
(`all_projects`, or the specific project IDs).

## Architecture

This is a **backend** data source plugin (Go binary + React editors), modeled on
[grafana/sentry-datasource](https://github.com/grafana/sentry-datasource):

- `QueryData` dispatches on `queryType` (`buildDuration` / `testDuration`) and
  calls `…/builds/metrics/duration` or `…/tests/metrics/duration`, then converts
  the `DurationMetrics` response into a wide time-series frame.
- `CallResource` proxies `GET /api/projects` and the `…/metrics/schemes` /
  `…/metrics/configurations` endpoints so the query editor dropdowns and template
  variables resolve without exposing the token.
- `CheckHealth` validates the token by listing projects.

See [AGENTS.md](./AGENTS.md) for the server-side API contract.

## Building

### Backend (Go)

```bash
mage -v            # build backend binaries for all platforms (gpx_tuist_datasource_*)
go test ./...      # run backend unit tests
```

### Frontend (Node pinned to 24 via `mise.toml`)

The webpack/jest/eslint harness under `.config/` is generated by
`@grafana/create-plugin` and is committed. Build and check with:

```bash
npm install
npm run typecheck
npm run lint
npm run build      # outputs dist/
```

`.config/` is managed by create-plugin — update it with
`npx @grafana/create-plugin@latest update` rather than editing it by hand.

### Run locally

```bash
mage -v && npm run build
npm run server     # docker compose with Grafana + this plugin mounted
```
