# Localization Context

This file provides context to the AI translation system (TranslateGemma) for translating Tuist's user-facing content.

## Product Overview

Tuist is a command-line tool and web platform for iOS/macOS developers that helps manage Xcode projects at scale. It provides:

- **Binary caching**: Cache and share build artifacts to speed up builds
- **App previews**: Deploy and share app builds for testing
- **Build analytics**: Track and analyze build performance
- **Swift Package Registry**: Host and manage Swift packages
- **Agentic QA**: AI-powered automated app testing

## Target Audience

Software developers working with Apple platforms (iOS, macOS, tvOS, watchOS, visionOS). Technical users familiar with Xcode, Swift, and command-line tools.

## Tone and Voice

- Professional but approachable
- Technical accuracy is paramount
- Concise and clear
- Developer-friendly (not marketing-heavy)
- Avoid corporate jargon

## Terminology Glossary

These terms have specific meanings in Tuist and should be translated carefully:

| English Term | Notes |
|--------------|-------|
| Cache | Binary cache for build artifacts, not "memory cache" |
| Bundle | Build artifact bundle (not "bundle" as in package) |
| Preview | App preview deployment feature |
| Registry | Swift Package Registry |
| Target | Xcode build target |
| Scheme | Xcode scheme |
| Workspace | Xcode workspace |
| Manifest | Tuist configuration file (Project.swift, etc.) |
| Graph | Project dependency graph |
| Agentic QA | AI-powered automated testing feature |
| Air | Tuist's subscription tier name (keep as "Air") |

## Do NOT Translate

- **Product names**: Tuist, Xcode, Swift, Apple, iOS, macOS, tvOS, watchOS, visionOS
- **Tier names**: Air (subscription tier)
- **Technical identifiers**: API endpoints, CLI flags, environment variables
- **Code snippets and examples**
- **Currency symbols**: Keep EUR, USD, etc. as-is
- **URLs and email addresses**
- **Numbers and statistics** (e.g., "10K+", "1400+")

## Placeholders

Preserve these placeholder formats exactly:
- `%{variable_name}` - Elixir/Phoenix interpolation
- `{{variable}}` - Alternative interpolation format

## Language-Specific Guidelines

### Japanese (ja)
- Use polite form (desu/masu style)
- Prefer katakana for technical terms when no standard Japanese term exists
- Keep product names in their original form

### Chinese Simplified (zh_Hans)
- Target mainland China audience
- Use simplified characters throughout
- Technical terms can use established Chinese equivalents when they exist

### Chinese Traditional (zh_Hant)
- Target Taiwan/Hong Kong audience
- Use traditional characters throughout

### Cantonese (yue_Hant)
- Use traditional characters
- Use Cantonese-specific vocabulary where appropriate

### Korean (ko)
- Use formal politeness level (hapnida style)
- Technical terms can remain in English when commonly used

### Arabic (ar)
- Right-to-left text direction
- Use Modern Standard Arabic

### Spanish (es)
- Use neutral Latin American Spanish where possible
- Avoid region-specific idioms

### Portuguese (pt)
- Use Brazilian Portuguese
- Avoid Portugal-specific expressions

### Russian (ru)
- Use formal "you" when addressing users

### Turkish (tr)
- Use formal register

### Polish (pl)
- Use formal register
