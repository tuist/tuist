# Редактирование {#editing}

В отличие от традиционных проектов Xcode или пакетов Swift, где изменения
вносятся через пользовательский интерфейс Xcode, проекты, управляемые Tuist,
определяются в коде Swift, содержащемся в манифест-файлах **** . Если вы знакомы
с пакетами Swift и файлом `Package.swift`, то этот подход очень похож.

Вы можете редактировать эти файлы с помощью любого текстового редактора, но мы
рекомендуем использовать для этого рабочий процесс, предоставляемый Tuist:
`tuist edit`. Этот рабочий процесс создает проект Xcode, который содержит все
манифестные файлы и позволяет вам редактировать и компилировать их. Благодаря
использованию Xcode вы получаете все преимущества автодополнения кода **,
подсветки синтаксиса и проверки ошибок**.

## Редактирование проекта {#edit-the-project}

Чтобы отредактировать свой проект, вы можете запустить следующую команду в
каталоге проекта Tuist или в подкаталоге:

```bash
tuist edit
```

Команда создает проект Xcode в глобальном каталоге и открывает его в Xcode.
Проект включает в себя каталог `Manifests`, который можно скомпилировать, чтобы
убедиться, что все манифесты действительны.

::: info GLOB-RESOLVED MANIFESTS
<!-- -->
`tuist edit` разрешает манифесты, которые должны быть включены, с помощью
глобального выражения `**/{Manifest}.swift` из корневого каталога проекта (того,
который содержит файл `Tuist.swift` ). Убедитесь, что в корневом каталоге
проекта есть действительный файл `Tuist.swift`.
<!-- -->
:::

### Игнорирование файлов манифеста {#ignoring-manifest-files}

Если ваш проект содержит файлы Swift с тем же именем, что и файлы манифеста
(например, `Project.swift`) в подкаталогах, которые не являются фактическими
манифестами Tuist, вы можете создать файл `.tuistignore` в корневом каталоге
вашего проекта, чтобы исключить их из редактируемого проекта.

Файл `.tuistignore` использует шаблоны glob для указания файлов, которые следует
игнорировать:

```gitignore
# Ignore all Project.swift files in the Sources directory
Sources/**/Project.swift

# Ignore specific subdirectories
Tests/Fixtures/**/Workspace.swift
```

Это особенно полезно, когда у вас есть тестовые фикстуры или примеры кода, в
которых используется та же конвенция именования, что и в манифест-файлах Tuist.

## Редактируйте и создавайте рабочий процесс {#edit-and-generate-workflow}

Как вы, наверное, заметили, редактирование не может быть выполнено из
сгенерированного проекта Xcode. Это сделано специально, чтобы сгенерированный
проект не зависел от Tuist, что гарантирует вам возможность перехода с Tuist в
будущем без особых усилий.

При итерации проекта мы рекомендуем запустить `tuist edit` из терминала, чтобы
получить проект Xcode для редактирования, и использовать другой терминал для
запуска `tuist generate`.
