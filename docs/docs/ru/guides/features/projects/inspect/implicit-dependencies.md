# Неявные импорты {#implicit-imports}

Чтобы упростить поддержание графа проекта Xcode с исходным проектом Xcode, Apple
разработала систему сборки, которая позволяет неявно определять зависимости. Это
означает, что продукт, например приложение, может зависеть от фреймворка, даже
без явного объявления зависимости. В небольших масштабах это не представляет
проблемы, но по мере усложнения графа проекта неявность может проявиться в виде
ненадежных инкрементальных сборок или функций редактора, таких как
предварительный просмотр или автодополнение кода.

Проблема заключается в том, что невозможно предотвратить возникновение неявных
зависимостей. Любой разработчик может добавить в свой код Swift оператор import`
` , и неявная зависимость будет создана. Именно здесь на помощь приходит Tuist.
Tuist предоставляет команду для проверки неявных зависимостей путем статического
анализа кода в вашем проекте. Следующая команда выведет неявные зависимости
вашего проекта:

```bash
tuist inspect dependencies --only implicit
```

Если команда обнаруживает какие-либо неявные импорты, она завершается с кодом
завершения, отличным от нуля.

::: tip VALIDATE IN CI
<!-- -->
Мы настоятельно рекомендуем запускать эту команду как часть вашей команды
<LocalizedLink href="/guides/features/automate/continuous-integration">непрерывной
интеграции</LocalizedLink> каждый раз, когда новый код отправляется вверх по
потоку.
<!-- -->
:::

::: warning NOT ALL IMPLICIT CASES ARE DETECTED
<!-- -->
Поскольку Tuist использует статический анализ кода для обнаружения неявных
зависимостей, он может не уловить все случаи. Например, Tuist не может понять
условные импорты через директивы компилятора в коде.
<!-- -->
:::
