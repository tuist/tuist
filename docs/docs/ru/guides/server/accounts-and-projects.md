# Учетные записи и проекты {#accounts-and-projects}

Некоторые функции Tuist требуют сервера, который обеспечивает постоянство данных
и может взаимодействовать с другими службами. Для взаимодействия с сервером вам
понадобится учетная запись и проект, который вы подключите к своему локальному
проекту.

## Учетные записи {#accounts}

Для использования сервера вам понадобится учетная запись. Существует два типа
учетных записей:

- **Личный аккаунт:** Эти аккаунты создаются автоматически при регистрации и
  идентифицируются по дескриптору, полученному либо от поставщика
  идентификационных данных (например, GitHub), либо из первой части адреса
  электронной почты.
- **Учетная запись организации:** Эти учетные записи создаются вручную и
  идентифицируются с помощью дескриптора, определенного разработчиком.
  Организации позволяют приглашать других членов для совместной работы над
  проектами.

Если вы знакомы с [GitHub](https://github.com), то концепция аналогична их
концепции, где вы можете иметь личные и организационные учетные записи, которые
идентифицируются с помощью дескриптора ** , который используется при построении
URL-адресов.

::: info CLI-FIRST
<!-- -->
Большинство операций по управлению учетными записями и проектами выполняется
через CLI. Мы работаем над веб-интерфейсом, который упростит управление учетными
записями и проектами.
<!-- -->
:::

Вы можете управлять организацией с помощью подкоманд в
<LocalizedLink href="/cli/organization">`tuist organization`</LocalizedLink>.
Чтобы создать новую учетную запись организации, выполните:
```bash
tuist organization create {account-handle}
```

## Проекты {#projects}

Ваши проекты, будь то Tuist или сырой Xcode, должны быть интегрированы с вашей
учетной записью через удаленный проект. Продолжая сравнение с GitHub, это похоже
на наличие локального и удаленного репозитория, куда вы отправляете свои
изменения. Вы можете использовать <LocalizedLink href="/cli/project">`tuist
project`</LocalizedLink> для создания и управления проектами.

Проекты идентифицируются полным дескриптором, который является результатом
объединения дескриптора организации и дескриптора проекта. Например, если у вас
есть организация с дескриптором `tuist` и проект с дескриптором `tuist`, полный
дескриптор будет `tuist/tuist`.

Связь между локальным и удаленным проектом осуществляется через файл
конфигурации. Если у вас его нет, создайте его по адресу `Tuist.swift` и
добавьте следующее содержимое:

```swift
let tuist = Tuist(fullHandle: "{account-handle}/{project-handle}") // e.g. tuist/tuist
```

::: warning TUIST PROJECT-ONLY FEATURES
<!-- -->
Обратите внимание, что некоторые функции, такие как
<LocalizedLink href="/guides/features/cache">бинарное
кэширование</LocalizedLink>, требуют наличия проекта Tuist. Если вы используете
необработанные проекты Xcode, вы не сможете использовать эти функции.
<!-- -->
:::

URL вашего проекта создается с использованием полного дескриптора. Например,
общедоступная панель управления Tuist доступна по адресу
[tuist.dev/tuist/tuist](https://tuist.dev/tuist/tuist), где `tuist/tuist`
является полным дескриптором проекта.
