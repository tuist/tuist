# Аутентификация {#authentication}

Для взаимодействия с сервером CLI необходимо аутентифицировать запросы с помощью
[аутентификации
bearer](https://swagger.io/docs/specification/authentication/bearer-authentication/).
CLI поддерживает аутентификацию в качестве пользователя, учетной записи или с
использованием токена OIDC.

## Как пользователь {#as-a-user}

При использовании CLI локально на вашем компьютере мы рекомендуем пройти
аутентификацию как пользователь. Для этого необходимо выполнить следующую
команду:

```bash
tuist auth login
```

Команда переведет вас через процесс веб-аутентификации. После аутентификации CLI
сохранит долгосрочный токен обновления и краткосрочный токен доступа в папке
`~/.config/tuist/credentials`. Каждый файл в этой папке представляет домен, на
котором вы прошли аутентификацию, который по умолчанию должен быть
`tuist.dev.json`. Информация, хранящаяся в этой папке, является
конфиденциальной, поэтому **убедитесь, что она находится в безопасности**.

CLI автоматически будет искать учетные данные при отправке запросов на сервер.
Если срок действия токена доступа истек, CLI будет использовать токен обновления
для получения нового токена доступа.

## Токены OIDC {#oidc-tokens}

Для сред CI, поддерживающих OpenID Connect (OIDC), Tuist может выполнять
аутентификацию автоматически, не требуя от вас управления долгосрочными
секретами. При работе в поддерживаемой среде CI CLI автоматически обнаружит
поставщика токенов OIDC и обменяет токен, предоставленный CI, на токен доступа
Tuist.

### Поддерживаемые поставщики CI {#supported-ci-providers}

- GitHub Actions
- CircleCI
- Bitrise

### Настройка аутентификации OIDC {#setting-up-oidc-authentication}

1. **Подключите свой репозиторий к Tuist**: Следуйте
   <LocalizedLink href="/guides/integrations/gitforge/github">руководству по
   интеграции с GitHub</LocalizedLink>, чтобы подключить свой репозиторий GitHub
   к проекту Tuist.

2. **Запустите `tuist auth login`**: В вашем рабочем процессе CI запустите
   `tuist auth login` перед любыми командами, требующими аутентификации. CLI
   автоматически определит среду CI и выполнит аутентификацию с помощью OIDC.

См.
<LocalizedLink href="/guides/integrations/continuous-integration">Руководство по
непрерывной интеграции</LocalizedLink> для примеров конфигурации для конкретных
провайдеров.

### Области действия токенов OIDC {#oidc-token-scopes}

Токенам OIDC предоставляется область действия `ci`, которая обеспечивает доступ
ко всем проектам, связанным с репозиторием. См. [Области
действия](#scope-groups) для получения подробной информации о том, что включает
в себя область действия `ci`.

::: tip SECURITY BENEFITS
<!-- -->
Аутентификация OIDC более безопасна, чем долгосрочные токены, потому что:
- Нет секретов, которые нужно раскрывать или управлять
- Токены имеют кратковременную жизнь и действуют только в рамках отдельных
  запусков рабочего процесса
- Аутентификация привязана к вашей идентичности в репозитории
<!-- -->
:::

## Токены учетной записи {#account-tokens}

Для сред CI, которые не поддерживают OIDC, или когда требуется точный контроль
над разрешениями, можно использовать токены учетной записи. Токены учетной
записи позволяют точно указать, к каким областям и проектам токен имеет доступ.

### Создание токена учетной записи {#creating-an-account-token}

```bash
tuist account tokens create my-account \
  --scopes project:cache:read project:cache:write \
  --name ci-cache-token \
  --expires 1y
```

Команда поддерживает следующие опции:

| Параметр    | Описание                                                                                                                                                  |
| ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--области` | Обязательно. Разделенный запятыми список областей, которым будет предоставлен токен.                                                                      |
| `--name`    | Обязательно. Уникальный идентификатор токена (1–32 символа, только буквы, цифры, дефисы и подчеркивания).                                                 |
| `--expires` | Необязательно. Когда токен должен истечь. Используйте формат типа `30d` (дни), `6m` (месяцы) или `1y` (годы). Если не указано, токен никогда не истекает. |
| `--проекты` | Ограничьте токен конкретными дескрипторами проектов. Если не указано иное, токен имеет доступ ко всем проектам.                                           |

### Доступные области {#available-scopes}

| Область применения       | Описание                              |
| ------------------------ | ------------------------------------- |
| `account:members:read`   | Прочитать участников учетной записи   |
| `account:members:write`  | Управление участниками аккаунта       |
| `account:registry:read`  | Читать из реестра Swift packages      |
| `account:registry:write` | Опубликовать в реестре Swift packages |
| `project:previews:read`  | Скачать превью                        |
| `project:previews:write` | Предварительный просмотр загрузки     |
| `project:admin:read`     | Прочитать настройки проекта           |
| `project:admin:write`    | Управление настройками проекта        |
| `project:cache:read`     | Скачать кэшированные бинарные файлы   |
| `project:cache:write`    | Загрузить кэшированные бинарные файлы |
| `project:bundles:read`   | Просмотреть наборы                    |
| `project:bundles:write`  | Загрузить пакеты                      |
| `project:tests:read`     | Прочитать результаты теста            |
| `project:tests:write`    | Загрузить результаты теста            |
| `project:builds:read`    | Прочитать аналитику сборки            |
| `project:builds:write`   | Загрузить аналитику сборки            |
| `project:runs:read`      | Команда «Read» выполняется            |
| `project:runs:write`     | Создание и обновление запусков команд |

### Группы областей действия {#scope-groups}

Группы областей действия предоставляют удобный способ присвоения нескольких
связанных областей действия с помощью одного идентификатора. При использовании
группы областей действия она автоматически расширяется, включая все отдельные
области действия, которые она содержит.

| Группа Scope | Включенные области                                                                                                                            |
| ------------ | --------------------------------------------------------------------------------------------------------------------------------------------- |
| `ci`         | `project:cache:write`, `project:previews:write`, `project:bundles:write`, `project:tests:write`, `project:builds:write`, `project:runs:write` |

### Непрерывная интеграция {#continuous-integration}

Для сред CI, не поддерживающих OIDC, вы можете создать токен учетной записи с
группой областей `ci` для аутентификации ваших рабочих процессов CI:

```bash
tuist account tokens create my-account --scopes ci --name ci
```

Это создает токен со всеми областями действия, необходимыми для типичных
операций CI (кеш, предварительный просмотр, пакеты, тесты, сборки и запуски).
Сохраните сгенерированный токен в качестве секрета в вашей среде CI и установите
его в качестве переменной среды `TUIST_TOKEN`.

### Управление токенами учетной записи {#managing-account-tokens}

Чтобы перечислить все токены для учетной записи:

```bash
tuist account tokens list my-account
```

Чтобы отозвать токен по имени:

```bash
tuist account tokens revoke my-account ci-cache-token
```

### Использование токенов учетной записи {#using-account-tokens}

Ожидается, что токены учетной записи будут определены как переменные среды
`TUIST_TOKEN`:

```bash
export TUIST_TOKEN=your-account-token
```

::: tip WHEN TO USE ACCOUNT TOKENS
<!-- -->
Используйте токены учетной записи, когда это необходимо:
- Аутентификация в средах CI, не поддерживающих OIDC
- Точный контроль над тем, какие операции может выполнять токен
- Токен, который может получить доступ к нескольким проектам в рамках одной
  учетной записи
- Временные токены, срок действия которых истекает автоматически
<!-- -->
:::
