# Регистрация {#logging}

CLI использует интерфейс [swift-log](https://github.com/apple/swift-log) для
ведения журнала. Пакет абстрагирует детали реализации ведения журнала, позволяя
CLI быть независимым от бэкэнда ведения журнала. Логгер вводится с помощью
локальных переменных задачи и доступен из любого места с помощью:

```bash
Logger.current
```

::: info
<!-- -->
Локальные переменные задачи не передают значение при использовании `Dispatch`
или отдельных задач, поэтому, если вы их используете, вам нужно будет получить
его и передать в асинхронную операцию.
<!-- -->
:::

## Что записывать в журнал {#what-to-log}

Журналы не являются пользовательским интерфейсом CLI. Они являются инструментом
для диагностики проблем, когда они возникают. Поэтому чем больше информации вы
предоставите, тем лучше. При создании новых функций поставьте себя на место
разработчика, столкнувшегося с неожиданным поведением, и подумайте, какая
информация будет ему полезна. Убедитесь, что вы используете правильный [уровень
журнала](https://www.swift.org/documentation/server/guides/libraries/log-levels.html).
В противном случае разработчики не смогут отфильтровать лишнюю информацию.
