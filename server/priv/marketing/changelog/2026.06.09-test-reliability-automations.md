Some tests are not flaky in the classic sense; they are just unhealthy, failing almost every time they run. Automations can now monitor **Test reliability**, the same success-rate metric shown on the test case detail page, and trigger when it drops below your chosen threshold. Pair it with actions like marking the test as flaky, changing its quarantine state, or notifying Slack, so consistently failing tests do not need to look flaky before Tuist can help clean them up.
