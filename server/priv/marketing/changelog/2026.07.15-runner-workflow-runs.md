The Tuist Runners dashboard now groups your CI by workflow, run, and job. Each workflow lists its runs, and every run opens a detail that rolls the whole run up: status and conclusion across all its jobs, total duration, branch, commit, and each job's own platform, status, and duration. Reruns collapse to their latest attempt, so a retried run reads as its current result, not its first.

When the Tuist GitHub App has write access to Actions, you can cancel a running workflow run straight from its detail, which cancels all of its jobs on GitHub.
