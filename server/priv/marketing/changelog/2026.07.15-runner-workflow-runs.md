The Tuist Runners dashboard now follows the same shape as your CI: work is grouped by workflow, each workflow lists its runs, and every run opens a detail view. No more scanning a flat list of jobs to piece together what happened.

The run detail rolls the whole run up into one place. It shows the run's status and conclusion derived from every job, the total duration, the branch and commit that triggered it, and the jobs that made it up, each with its own platform, status, and duration. Reruns collapse to their latest attempt, so a run that failed and was retried reads as its current result rather than its first.

When the Tuist GitHub App has write access to Actions, you can also cancel a running workflow run straight from the run detail, which cancels all of its jobs on GitHub.
