If your network routes outbound traffic through an HTTP proxy, you can now configure Tuist to route its client-side HTTP connections through it. The setting is available in `Tuist.swift` for Xcode projects, in `settings.gradle.kts` for Gradle projects via the Tuist Gradle plugin, and in `tuist.toml` when you want the same proxy to apply to both the CLI and the Gradle plugin. It applies to every HTTP client Tuist manages (cache, previews, analytics, registry access, and the calls the Gradle plugin makes back to Tuist services) and leaves the build systems Tuist integrates with untouched. See the [HTTP proxy integration guide](/guides/integrations/http-proxy) for the full configuration details.
