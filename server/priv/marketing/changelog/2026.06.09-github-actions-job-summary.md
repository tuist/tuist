When Tuist runs in GitHub Actions, the same Tuist Run Report that the GitHub app posts as a pull request comment now also appears in the Actions job summary. The test, build, and bundle results for the run show up right on the workflow run page, with nothing extra to set up.

This is especially useful with merge queues. A merge queue run is not attached to a pull request, so there is no comment to post, and until now you had to wire up your own JUnit reporting to see results. The report now shows up in the job summary for every authenticated run, whether it came from a pull request or a queue.
