---
title: SwifterPM is now the default for generated projects
description: "tuist install now resolves and restores Swift packages with SwifterPM by default, with TUIST_DISABLE_SWIFTERPM=1 as an opt-out."
category: "Product"
---

`tuist install` now uses [SwifterPM](https://github.com/tuist/tuist/tree/main/swifterpm) by default to fetch and restore your Swift package dependencies. Resolution stays delegated to the Swift Package Manager, while restoration links checkouts back to a global content-addressable store instead of copying them into every worktree, making warm restores sub-second and saving gigabytes of disk across worktrees.

Until now this was opt-in through `TUIST_USE_SWIFTERPM=1`. That variable is no longer needed. If you hit a package graph that SwifterPM does not handle yet, you can fall back to SwiftPM by setting `TUIST_DISABLE_SWIFTERPM=1` in your environment, and please [open an issue](https://github.com/tuist/tuist/issues) so we can fix it.
