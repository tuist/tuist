Every Tuist account now exposes a Prometheus-compatible scrape endpoint at `/api/accounts/:handle/metrics` with per-account counters and histograms covering Xcode and Gradle builds, test runs, binary cache events, and CLI invocations. Point Grafana Alloy, Grafana Agent, or any Prometheus-compatible scraper at it with an account token carrying the `account:metrics:read` scope, and your existing observability stack can query Tuist telemetry alongside the rest of your infrastructure.

```bash
tuist account tokens create <your-handle> \
  --scopes account:metrics:read \
  --name grafana
```

See the [metrics reference](https://tuist.dev/en/docs/references/metrics) for the full vocabulary of metric names, label sets, and histogram buckets.
