Tuist now integrates with self-hosted GitHub Enterprise Server (GHES) instances. Switch to the **Enterprise server** tab on your organization's integrations page, enter your GHES base URL, and click install — Tuist routes you through GitHub's [App manifest flow](https://docs.github.com/en/apps/sharing-github-apps/registering-a-github-app-from-a-manifest), which registers a fresh Tuist GitHub App on your GHES instance and hands the App's credentials back to Tuist. From there, comments, check runs, and repository connections all flow through your private GHES.

GitHub Apps are scoped to a single GitHub instance, so the github.com Tuist App can't be installed on a GHES instance — every customer registers their own copy. The credentials Tuist receives from the manifest exchange are stored encrypted per-installation. No env vars or Tuist server configuration changes are needed.
