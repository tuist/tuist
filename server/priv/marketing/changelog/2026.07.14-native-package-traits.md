---
title: Swift package traits in native integration
description: "Select Swift package traits directly alongside native package declarations in a Tuist project manifest."
category: "Product"
pull_request: 11799
---

Tuist projects that use Xcode's native package integration can now select Swift package traits directly in `Project.packages`:

```swift
let project = Project(
    name: "App",
    packages: [
        .package(path: "Package", traits: ["NativeIntegration"]),
    ]
)
```

Package traits require Xcode 26.4 or later. Omitting `traits` keeps the package's default traits, while passing an empty array disables all default traits. Existing package declarations continue to work unchanged.
