## Description

### What problem or need do you have?

We currently don't generate resource bundle accessors for internal targets that purely include Objective C sources.
We used to do that but had to revert that in [this PR](https://github.com/tuist/tuist/pull/6388).

In other words, an objc accessor file should be generated for `Framework1` of the following project:
[ios_app_with_framework_and_disabled_resources.zip](https://github.com/user-attachments/files/16027873/ios_app_with_framework_and_disabled_resources.zip)



### Potential solution

As described in the [PR](https://github.com/tuist/tuist/pull/6388), before we can [delete the line](https://github.com/tuist/tuist/blob/main/Sources/TuistGenerator/Mappers/ResourcesProjectMapper.swift#L84) guarding the objc resource accessors only to external projects, we will need to figure out how we can generate the Objc resource accessor without overriding the `GCC_PREFIX_HEADER`.

4.19.0
