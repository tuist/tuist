Goal: Recreate PR #8423 changes on current repo: move build_runs fully to ClickHouse, remove all Timescale usage, insert via ingestion buffers, and migrate existing data.

Context and constraints
- Current build_runs live in Postgres via `Tuist.Builds.Build` and `Tuist.Builds` (writes use `Repo.insert`, reads use `Repo` and Flop).
- Build analytics in `Tuist.Builds.Analytics` use `Timescale.Hyperfunctions.time_bucket`.
- ClickHouse ingestion is the standard pattern (RowBinary buffers in `Tuist.Ingestion.Bufferable` + buffer modules like `Tuist.Xcode.XcodeGraph.Buffer`).
- Requirement: remove Timescale dependency (extension + time_bucket usage). Keep Postgres build_runs table read-only for rollback (per user).

Recommended plan

1) Create ClickHouse build_runs table + backfill migration
   - Add ClickHouse migration at `server/priv/ingest_repo/migrations/*_create_build_runs.exs`.
   - Table schema should mirror current Postgres build_runs fields, using ClickHouse types:
     - id UUID, duration Int32/Int64, project_id Int64, account_id Int64, is_ci Bool, inserted_at DateTime/DateTime64.
     - enums as ClickHouse Enum8 with string values (status/category/ci_provider) to align with Ecto ClickHouse adapter behavior.
     - custom_tags Array(String), custom_values Map(String, String) (or Map(String, String) / Nested per adapter support).
   - Include backfill using ClickHouse `postgresql()` table function selecting from Postgres build_runs.
     - Map enum ints to strings via `multiIf`/`transform`.
     - Ensure null handling with `ifNull` for optional fields.
     - Use a single insert-select in the migration’s `up`.

2) Move build_runs schema to ClickHouse + enable ingestion buffering
   - Update `server/lib/tuist/builds/build.ex`:
     - Switch to ClickHouse schema (use `Ch` types, `@primary_key {:id, Ch, type: "UUID"}` or `UUIDv7.generate` in changeset).
     - Add `use Tuist.Ingestion.Bufferable` and remove Postgres-specific validations/constraints that rely on Repo.
     - Ensure enum normalization for status/category/ci_provider to string values compatible with ClickHouse Enum8.
   - Add buffer module similar to `Tuist.Xcode.XcodeGraph.Buffer`:
     - New file: `server/lib/tuist/builds/buffer/build.ex` with RowBinary insert for build_runs.
   - Optionally align build_files/build_issues/build_targets with buffer ingestion (as in PR #8423):
     - Add `server/lib/tuist/builds/buffer/build_file.ex`, `build_issue.ex`, `build_target.ex`.
     - Update writes to use buffers instead of `IngestRepo.insert_all`.

3) Switch build_runs write and read paths to ClickHouse
   - Update `server/lib/tuist/builds.ex`:
     - `create_build/1`: build changeset -> struct -> `Build.Buffer.insert/1` (ingestion pattern).
     - Replace `Repo` usage for reads with `ClickHouseRepo` (or `ClickHouseFlop` for pagination) where appropriate:
       - `get_build/1`, `list_build_runs/2`, `project_build_schemes/1`, `project_build_configurations/1`, `project_build_tags/1`.
     - For preloads, load account/project info via separate Repo queries (ClickHouse can’t preload from Postgres).

4) Replace Timescale analytics with ClickHouse-native queries
   - Update `server/lib/tuist/builds/analytics.ex`:
     - Remove `import Timescale.Hyperfunctions` and `time_bucket` usage.
     - Use ClickHouse `formatDateTime` / `toStartOfInterval` with date range builders (pattern from `Tuist.CommandEvents` in PR #8423).
     - Ensure all analytics that currently use Repo queries move to ClickHouseRepo queries.
   - Remove or refactor any remaining `time_bucket` helpers in builds analytics.

5) Remove Timescale dependency and references
   - Remove `{:timescale, "~> 0.1.0"}` from `server/mix.exs`.
   - Remove `Tuist.Repo.timescale_available?/0` and all usages:
     - `server/lib/tuist_web/plugs/redirect_to_runs_plug.ex` (drop plug logic or always allow builds page).
     - Update tests stubbing `timescale_available?`.
   - Update Postgres migrations that call `create_hypertable` to avoid Timescale calls (edit to no-op in new setups):
     - `server/priv/repo/migrations/20240512093328_add_command_events_hypertable.exs`
     - `server/priv/repo/migrations/20250224154433_add_build_runs_table.exs`
     - Any timescale-specific comments or guards in other migrations.

6) Data-export documentation update
   - Update `server/data-export.md` to document:
     - build_runs now stored in ClickHouse (table name, key fields, custom tags/values, CI metadata, git metadata).
     - Retention/relationships with build_issues, build_files, build_targets, cacheable_tasks, cas_outputs.

7) Tests and verification updates
